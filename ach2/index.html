2. 정렬(Sorting)이란?

정렬은 데이터를 특정 기준에 따라 순서대로 나열하는 것입니다. 정보올림피아드에서는 정렬을 통해 탐색, 비교, 중복 제거, 조건 판단을 훨씬 쉽게 만들 수 있습니다.

예시

원래 데이터: 5 1 3 2 4

오름차순: 1 2 3 4 5

내림차순: 5 4 3 2 1

2.1 정렬이 중요한 이유

  1. 탐색 속도를 빠르게 만들 수 있습니다.
  2. 이진 탐색을 사용할 수 있습니다.
  3. 중복 제거가 쉬워집니다.
  4. 조건 비교가 쉬워집니다.
  5. 구간 처리 문제에 활용할 수 있습니다.
정렬은 단독으로도 중요하지만, 보통 이진 탐색이나 투 포인터 같은 알고리즘과 함께 사용될 때 더 강력해집니다.

3. 대표 정렬 알고리즘

정렬 평균 시간복잡도 특징
버블 정렬O(N²)구현이 쉽지만 느림
선택 정렬O(N²)가장 작은 값을 선택
삽입 정렬O(N²)거의 정렬된 경우 빠름
병합 정렬O(N log N)안정 정렬, 분할 정복
퀵 정렬O(N log N)평균적으로 매우 빠름
qsort / sortO(N log N)실전에서 가장 많이 사용

3.1 버블 정렬

인접한 두 수를 비교해서 큰 값을 뒤로 보내는 방식입니다.

5 3 2 4 3 5 2 4 3 2 5 4 3 2 4 5 
#include <stdio.h>

int main() {
    int arr[5] = {5, 3, 2, 4, 1};

    for (int i = 0; i < 5; i++) {
        for (int j = 0; j < 4 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }

    for (int i = 0; i < 5; i++) {
        printf("%d ", arr[i]);
    }

    return 0;
}

3.2 선택 정렬

남은 숫자 중 가장 작은 값을 찾아 앞쪽으로 보내는 방식입니다.

5 1 4 2 3 1 5 4 2 3 1 2 4 5 3 1 2 3 5 4
선택 정렬은 구현은 쉽지만 매번 가장 작은 값을 찾기 때문에 시간복잡도는 O(N²)입니다.

3.3 삽입 정렬

현재 숫자를 앞쪽의 이미 정렬된 부분에 알맞게 끼워 넣는 방식입니다. 카드를 손에 들고 정렬하는 방식과 비슷합니다.

삽입 정렬은 거의 정렬된 데이터에서는 매우 빠르게 동작할 수 있습니다.

3.4 병합 정렬

배열을 반으로 나누고, 각각 정렬한 뒤 다시 합치는 분할 정복 방식입니다.

나누기 각각 정렬 합치기 완성
시간복잡도: O(N log N)

3.5 퀵 정렬

기준값 Pivot을 정하고, 작은 값은 왼쪽, 큰 값은 오른쪽으로 나누는 방식입니다.

퀵 정렬은 평균적으로 빠르지만, Pivot 선택이 나쁘면 최악의 경우 O(N²)이 될 수 있습니다.

4. 조건별 최적 정렬 선택

상황 추천 정렬 이유
데이터가 매우 적음버블 / 선택 / 삽입구현이 간단함
거의 정렬되어 있음삽입 정렬이동 횟수가 적음
일반적인 대량 데이터qsort / sortO(N log N)으로 빠름
안정 정렬이 필요함병합 정렬 / stable sort같은 값의 기존 순서 유지
실전 대회표준 라이브러리 정렬빠르고 검증되어 있음
중등정보올림피아드 실전에서는 직접 O(N²) 정렬을 구현하기보다 qsort, sort, 또는 문제 조건에 맞는 효율적인 정렬을 사용하는 것이 중요합니다.

5. C언어 qsort 사용법

C언어에서는 stdlib.h의 qsort를 사용해 배열을 정렬할 수 있습니다.

5.1 기본 정수 정렬

#include <stdio.h>
#include <stdlib.h>

int compare(const void *a, const void *b) {
    int x = *(int*)a;
    int y = *(int*)b;

    if (x < y) return -1;
    if (x > y) return 1;
    return 0;
}

int main() {
    int arr[5] = {5, 1, 4, 2, 3};

    qsort(arr, 5, sizeof(int), compare);

    for (int i = 0; i < 5; i++) {
        printf("%d ", arr[i]);
    }

    return 0;
}
compare 함수에서 단순히 return x - y;를 사용하면 값이 매우 클 때 오버플로우가 날 수 있습니다. 안전하게 if문으로 비교하는 습관이 좋습니다.

6. 좌표 정렬

6.1 문제 설명

좌표 (x, y)가 여러 개 주어질 때 다음 기준으로 정렬합니다.

  1. x 기준 오름차순
  2. x가 같으면 y 기준 오름차순

6.2 구조체 정의

typedef struct {
    int x;
    int y;
} Point;

6.3 비교 함수

int compare(const void *a, const void *b) {
    Point *p1 = (Point*)a;
    Point *p2 = (Point*)b;

    if (p1->x < p2->x) return -1;
    if (p1->x > p2->x) return 1;

    if (p1->y < p2->y) return -1;
    if (p1->y > p2->y) return 1;
    return 0;
}

6.4 전체 코드

#include <stdio.h>
#include <stdlib.h>

#define MAX 100000

typedef struct {
    int x;
    int y;
} Point;

Point arr[MAX];

int compare(const void *a, const void *b) {
    Point *p1 = (Point*)a;
    Point *p2 = (Point*)b;

    if (p1->x < p2->x) return -1;
    if (p1->x > p2->x) return 1;

    if (p1->y < p2->y) return -1;
    if (p1->y > p2->y) return 1;
    return 0;
}

int main() {
    int N;
    scanf("%d", &N);

    for (int i = 0; i < N; i++) {
        scanf("%d %d", &arr[i].x, &arr[i].y);
    }

    qsort(arr, N, sizeof(Point), compare);

    for (int i = 0; i < N; i++) {
        printf("%d %d\n", arr[i].x, arr[i].y);
    }

    return 0;
}

7. 나이순 정렬

7.1 문제 특징

나이순 정렬 문제는 보통 다음 조건을 가집니다.

  1. 나이 오름차순
  2. 나이가 같으면 입력 순서 유지
같은 값의 기존 순서를 유지하는 정렬을 안정 정렬이라고 합니다.

7.2 C언어에서 처리하는 방법

C의 qsort는 안정 정렬을 보장하지 않습니다. 따라서 입력 순서를 index로 함께 저장하고, 나이가 같으면 index를 기준으로 정렬합니다. 

#include <stdio.h>
#include <stdlib.h>

#define MAX 100000

typedef struct {
    int age;
    char name[101];
    int index;
} Member;

Member arr[MAX];

int compare(const void *a, const void *b) {
    Member *m1 = (Member*)a;
    Member *m2 = (Member*)b;

    if (m1->age < m2->age) return -1;
    if (m1->age > m2->age) return 1;

    if (m1->index < m2->index) return -1;
    if (m1->index > m2->index) return 1;
    return 0;
}

int main() {
    int N;
    scanf("%d", &N);

    for (int i = 0; i < N; i++) {
        scanf("%d %s", &arr[i].age, arr[i].name);
        arr[i].index = i;
    }

    qsort(arr, N, sizeof(Member), compare);

    for (int i = 0; i < N; i++) {
        printf("%d %s\n", arr[i].age, arr[i].name);
    }

    return 0;
}

8. 이진 탐색(Binary Search)

8.1 왜 필요한가?

배열에서 특정 값을 찾을 때 하나씩 찾으면 O(N)이 걸립니다. 하지만 배열이 정렬되어 있다면 중간값을 기준으로 탐색 범위를 절반씩 줄일 수 있습니다.

순차 탐색 앞에서부터 하나씩 확인합니다.
시간복잡도: O(N)
이진 탐색 중간을 기준으로 절반씩 줄입니다.
시간복잡도: O(log N)
이진 탐색은 반드시 정렬된 배열에서만 사용할 수 있습니다.

8.2 이진 탐색 원리

배열: 1 3 5 7 9 11 13

찾는 값: 9

중간값: 7

9는 7보다 크므로 오른쪽 부분만 다시 탐색합니다.

8.3 C 코드

#include <stdio.h>

int main() {
    int arr[7] = {1, 3, 5, 7, 9, 11, 13};
    int target = 9;

    int left = 0;
    int right = 6;

    while (left <= right) {
        int mid = (left + right) / 2;

        if (arr[mid] == target) {
            printf("찾음\n");
            return 0;
        }

        if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    printf("없음\n");
    return 0;
}
left와 right가 의미하는 범위를 정확히 유지하는 것이 이진 탐색 구현의 핵심입니다.

9. lower_bound와 upper_bound 개념

9.1 lower_bound

lower_bound는 어떤 값 이상이 처음 등장하는 위치를 찾는 개념입니다.

9.2 upper_bound

upper_bound는 어떤 값을 초과하는 첫 위치를 찾는 개념입니다.

배열: 1 2 2 2 3 4

값 2의 lower_bound: 첫 번째 2의 위치

값 2의 upper_bound: 3의 위치

값 2의 개수: upper_bound - lower_bound

9.3 lower_bound C 구현

int lower_bound(int arr[], int n, int target) {
    int left = 0;
    int right = n;

    while (left < right) {
        int mid = (left + right) / 2;

        if (arr[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }

    return left;
}

9.4 upper_bound C 구현

int upper_bound(int arr[], int n, int target) {
    int left = 0;
    int right = n;

    while (left < right) {
        int mid = (left + right) / 2;

        if (arr[mid] > target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }

    return left;
}

10. 매개 변수 탐색(Parametric Search)

10.1 핵심 개념

매개 변수 탐색은 정답을 직접 찾는 것이 아니라, 어떤 값이 가능한지 검사하면서 정답 범위를 이진 탐색하는 방법입니다.

정답 후보 X → 가능한가? → 가능하면 더 큰 값 / 불가능하면 더 작은 값

10.2 기본 흐름

  1. 정답이 될 수 있는 범위를 정합니다.
  2. 중간값 mid를 정합니다.
  3. mid가 가능한지 검사합니다.
  4. 가능하면 정답을 저장하고 더 좋은 방향으로 탐색합니다.
  5. 불가능하면 반대 방향으로 탐색합니다.

10.3 이런 문장이 나오면 의심하기

최대값가능한 값 중 가장 큰 값을 구하라.
최솟값조건을 만족하는 가장 작은 값을 구하라.
가능 여부이 값으로 조건을 만족할 수 있는가?
최적화가장 좋은 높이, 길이, 거리, 용량을 구하라.
매개 변수 탐색 문제는 대부분 “정답 후보를 정했을 때 가능 여부를 빠르게 판단할 수 있는가?”가 핵심입니다.

11. 랜선 자르기

11.1 문제 설명

여러 개의 랜선을 같은 길이로 잘라 N개 이상 만들고 싶습니다. 가능한 랜선 길이 중 최대값을 구하는 문제입니다.

11.2 핵심 아이디어

랜선의 길이를 기준으로 이진 탐색합니다.

길이 200 가능? 가능하면 더 길게 길이 300 가능? 불가능하면 더 짧게

11.3 C 코드

#include <stdio.h>

long long arr[10001];

int main() {
    int K, N;
    scanf("%d %d", &K, &N);

    long long max = 0;

    for (int i = 0; i < K; i++) {
        scanf("%lld", &arr[i]);
        if (arr[i] > max) {
            max = arr[i];
        }
    }

    long long left = 1;
    long long right = max;
    long long answer = 0;

    while (left <= right) {
        long long mid = (left + right) / 2;
        long long count = 0;

        for (int i = 0; i < K; i++) {
            count += arr[i] / mid;
        }

        if (count >= N) {
            answer = mid;
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    printf("%lld\n", answer);
    return 0;
}
길이가 길수록 만들 수 있는 랜선 개수는 줄어듭니다. 이런 단조성이 있기 때문에 이진 탐색이 가능합니다.

12. 나무 자르기

12.1 문제 설명

절단기 높이를 정해서 나무를 자릅니다. 잘린 나무 길이의 합이 필요한 길이 이상이 되도록 하는 최대 절단기 높이를 구합니다.

12.2 핵심 아이디어

절단기 높이를 기준으로 이진 탐색합니다.

절단기 높이가 낮으면 많이 잘립니다.

절단기 높이가 높으면 적게 잘립니다.

따라서 높이에 따라 얻는 나무 길이가 단조적으로 변합니다.

12.3 C 코드

#include <stdio.h>

#define MAX 1000000

long long tree[MAX];

int main() {
    int N;
    long long M;
    scanf("%d %lld", &N, &M);

    long long maxHeight = 0;

    for (int i = 0; i < N; i++) {
        scanf("%lld", &tree[i]);
        if (tree[i] > maxHeight) {
            maxHeight = tree[i];
        }
    }

    long long left = 0;
    long long right = maxHeight;
    long long answer = 0;

    while (left <= right) {
        long long mid = (left + right) / 2;
        long long sum = 0;

        for (int i = 0; i < N; i++) {
            if (tree[i] > mid) {
                sum += tree[i] - mid;
            }
        }

        if (sum >= M) {
            answer = mid;
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    printf("%lld\n", answer);
    return 0;
}

14. 핵심 정리

  1. 정렬은 데이터를 기준에 따라 순서대로 배치하는 과정입니다.
  2. 실전에서는 O(N log N) 정렬을 주로 사용합니다.
  3. 버블, 선택, 삽입 정렬은 구현은 쉽지만 일반적으로 O(N²)입니다.
  4. 병합 정렬과 퀵 정렬은 평균적으로 O(N log N)입니다.
  5. C언어에서는 qsort를 활용할 수 있습니다.
  6. 좌표 정렬은 x 기준, x가 같으면 y 기준으로 정렬합니다.
  7. 나이순 정렬은 나이가 같을 때 입력 순서를 유지해야 합니다.
  8. 이진 탐색은 반드시 정렬된 배열에서 사용합니다.
  9. 이진 탐색의 시간복잡도는 O(log N)입니다.
  10. 매개 변수 탐색은 “가능한가?”를 기준으로 정답 범위를 줄이는 방법입니다.
  11. 랜선 자르기와 나무 자르기는 대표적인 최댓값 최적화 문제입니다.
  12. 정렬 후 이진 탐색하는 패턴은 정보올림피아드에서 매우 자주 등장합니다.
