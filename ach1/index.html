시간복잡도와 기초 자료구조

알고리즘의 효율성 분석, 스택과 큐, 빠른 입출력, 올바른 괄호 쌍, 구간 연산과 큐 회전 시뮬레이션을 한 번에 정리하는 강의 노트입니다.

학습 목표 시간복잡도 빅오 표기법 스택 큐 입출력 최적화 괄호 문제 구간 연산 큐 회전 실전 문제 핵심 정리

1. 학습 목표

1. 알고리즘의 실행 시간이 입력 크기 N에 따라 어떻게 변하는지 이해한다.
2. O(1), O(N), O(N log N)의 차이를 설명할 수 있다.
3. 스택과 큐의 구조를 이해하고 문제 해결에 활용한다.
4. C언어와 Python에서 빠른 입출력을 사용하는 이유를 이해한다.
5. 올바른 괄호 쌍 문제를 스택으로 해결한다.
6. 수열의 구간 연산과 큐 회전 시뮬레이션을 구현한다.

2. 알고리즘의 효율성 분석

2.1 알고리즘이란?

알고리즘은 어떤 문제를 해결하기 위한 정해진 절차입니다. 정보올림피아드에서는 같은 문제라도 어떤 방법으로 해결하느냐에 따라 정답이 되기도 하고 시간초과가 되기도 합니다.

예: N개의 숫자의 합 구하기

sum = 0;
for (i = 0; i < N; i++) {
    sum += arr[i];
}

이 코드는 숫자 N개를 하나씩 모두 확인합니다. 입력이 10개면 10번, 100000개면 100000번 정도 실행됩니다.

시간복잡도 분석은 입력 크기가 커졌을 때 알고리즘이 버틸 수 있는지 판단하는 가장 중요한 기준입니다.

3. 빅오 표기법

빅오 표기법은 알고리즘의 실행 시간이 입력 크기 N에 따라 얼마나 빠르게 증가하는지 나타내는 방법입니다.

시간복잡도 의미 대표 예시
O(1) 입력 크기와 상관없이 일정한 시간 배열의 특정 위치 접근
O(N) 입력 크기만큼 반복 배열 전체 합 구하기
O(N log N) 정렬에서 자주 등장 빠른 정렬, 병합 정렬
O(N²) 모든 쌍을 비교 이중 반복문

3.1 O(1)

O(1)은 입력 크기와 관계없이 실행 시간이 거의 일정한 경우입니다. 

int arr[5] = {10, 20, 30, 40, 50};
printf("%d\n", arr[2]);

3.2 O(N)

O(N)은 입력 크기 N만큼 반복하는 알고리즘입니다. 

int sum = 0;
for (int i = 0; i < N; i++) {
    sum += arr[i];
}

3.3 O(N log N)

O(N log N)은 정렬 알고리즘에서 자주 등장합니다. C의 qsort, Python의 sort는 일반적으로 O(N log N) 정렬로 볼 수 있습니다.

N = 100000일 때 대략적인 연산 횟수
O(N) 100000
O(N log N) 약 1700000
O(N²) 10000000000
N이 100000 이상인데 이중 반복문을 사용하면 대부분 시간초과를 의심해야 합니다.

4. 스택(Stack)

스택은 나중에 들어온 데이터가 먼저 나가는 구조입니다. 이를 LIFO라고 합니다.

push 10 push 20 push 30 pop → 30 pop → 20 pop → 10
연산 의미
push스택에 데이터 넣기
pop스택에서 맨 위 데이터 꺼내기
top맨 위 데이터 확인하기
empty스택이 비었는지 확인하기

4.1 C언어로 스택 구현

#include <stdio.h>

#define MAX 100000

int stack[MAX];
int top = -1;

void push(int value) {
    stack[++top] = value;
}

int pop() {
    return stack[top--];
}

int peek() {
    return stack[top];
}

int empty() {
    return top == -1;
}

int main() {
    push(10);
    push(20);
    push(30);

    printf("%d\n", pop());
    printf("%d\n", pop());
    printf("%d\n", pop());

    return 0;
}

5. 큐(Queue)

큐는 먼저 들어온 데이터가 먼저 나가는 구조입니다. 이를 FIFO라고 합니다.

push 10 push 20 push 30 pop → 10 pop → 20 pop → 30
연산 의미
push / enqueue큐 뒤에 데이터 넣기
pop / dequeue큐 앞에서 데이터 꺼내기
front큐의 맨 앞 데이터 확인
empty큐가 비었는지 확인

5.1 C언어로 큐 구현

#include <stdio.h>

#define MAX 100000

int queue[MAX];
int front = 0;
int rear = 0;

void push(int value) {
    queue[rear++] = value;
}

int pop() {
    return queue[front++];
}

int empty() {
    return front == rear;
}

int main() {
    push(10);
    push(20);
    push(30);

    printf("%d\n", pop());
    printf("%d\n", pop());
    printf("%d\n", pop());

    return 0;
}

6. 표준 라이브러리와 입출력 최적화

정보올림피아드에서는 입력 데이터가 매우 많을 수 있습니다. 입력이 많을 때 느린 입출력을 사용하면 알고리즘이 맞아도 시간초과가 날 수 있습니다.

6.1 C언어 기본 입출력

scanf("%d", &x);
printf("%d\n", x);

중등 수준의 많은 문제는 scanf와 printf만으로 충분합니다.

6.2 C언어 빠른 입력 함수

#include <stdio.h>

int fastInput() {
    int x = 0;
    int c = getchar();

    while (c < '0' || c > '9') {
        c = getchar();
    }

    while (c >= '0' && c <= '9') {
        x = x * 10 + (c - '0');
        c = getchar();
    }

    return x;
}

6.3 Python 입출력 최적화

import sys
input = sys.stdin.readline

N = int(input())
arr = list(map(int, input().split()))
print(sum(arr))
Python에서 입력이 많으면 input() 대신 sys.stdin.readline을 사용하는 습관을 들이는 것이 좋습니다.

7. 대표 문제: 올바른 괄호 쌍 찾기

7.1 문제 설명

괄호 문자열이 주어졌을 때 올바른 괄호 문자열인지 판단합니다.

올바른 예: (())

잘못된 예: ())(

7.2 해결 아이디어

  1. 여는 괄호 '('가 나오면 스택에 넣습니다.
  2. 닫는 괄호 ')'가 나오면 스택에서 여는 괄호 하나를 꺼냅니다.
  3. 닫는 괄호가 나왔는데 스택이 비어 있으면 잘못된 문자열입니다.
  4. 문자열을 모두 확인한 뒤 스택이 비어 있어야 올바른 괄호 문자열입니다.

7.3 C 정답 코드

#include <stdio.h>
#include <string.h>

#define MAX 100001

char stack[MAX];
int top = -1;

int main() {
    char s[MAX];
    int valid = 1;

    scanf("%s", s);

    for (int i = 0; i < strlen(s); i++) {
        if (s[i] == '(') {
            stack[++top] = '(';
        } else {
            if (top == -1) {
                valid = 0;
                break;
            }
            top--;
        }
    }

    if (top != -1) {
        valid = 0;
    }

    if (valid) {
        printf("YES\n");
    } else {
        printf("NO\n");
    }

    return 0;
}
문자열 길이를 N이라고 하면 전체 시간복잡도는 O(N)입니다.

8. 수열의 구간 연산

8.1 구간 합 문제

수열에서 L번째부터 R번째까지의 합을 여러 번 구해야 하는 문제가 자주 등장합니다.

수열: 3 2 4 5 1

질문: 2번째부터 4번째까지의 합은?

답: 2 + 4 + 5 = 11

8.2 누적합

누적합은 앞에서부터 더한 값을 미리 저장해 두는 방법입니다. 

arr    = [3, 2, 4, 5, 1]
prefix = [0, 3, 5, 9, 14, 15]

구간 L부터 R까지의 합 = prefix[R] - prefix[L - 1]

8.3 C 정답 코드

#include <stdio.h>

#define MAX 100001

int prefix[MAX];

int main() {
    int N, Q;

    scanf("%d %d", &N, &Q);

    for (int i = 1; i <= N; i++) {
        int x;
        scanf("%d", &x);
        prefix[i] = prefix[i - 1] + x;
    }

    for (int i = 0; i < Q; i++) {
        int L, R;
        scanf("%d %d", &L, &R);

        printf("%d\n", prefix[R] - prefix[L - 1]);
    }

    return 0;
}
누적합을 만들면 각 질문을 O(1)에 처리할 수 있습니다. 전체 시간복잡도는 O(N + Q)입니다.

9. 큐 회전 시뮬레이션

큐 회전은 맨 앞의 값을 꺼내서 다시 뒤에 넣는 방식으로 구현합니다.

1 2 3 4 5 왼쪽 1회전 2 3 4 5 1 왼쪽 2회전 3 4 5 1 2

9.1 핵심 코드

int x = pop();
push(x);

9.2 C 정답 코드

#include <stdio.h>

#define MAX 200001

int queue[MAX];
int front = 0;
int rear = 0;

void push(int x) {
    queue[rear++] = x;
}

int pop() {
    return queue[front++];
}

int main() {
    int N, K;

    scanf("%d %d", &N, &K);

    for (int i = 1; i <= N; i++) {
        push(i);
    }

    for (int i = 0; i < K; i++) {
        int x = pop();
        push(x);
    }

    while (front < rear) {
        printf("%d ", pop());
    }

    return 0;
}

10. 실전 문제 유형

시간복잡도 판단 반복문 구조를 보고 O(1), O(N), O(N log N), O(N²)를 판단합니다.
스택 문제 괄호 검사, 되돌리기, 최근 데이터 처리, 막대기 문제 등에 활용합니다.
큐 문제 줄 서기, 카드 버리기, 회전 시뮬레이션, BFS 기초에 활용합니다.
구간 연산 누적합을 사용해 여러 번의 구간 합 질문에 빠르게 답합니다.

수업 중 확인 질문

  1. 배열의 특정 위치 값을 바로 가져오는 시간복잡도는 무엇인가?
  2. N개의 숫자를 모두 더하는 알고리즘의 시간복잡도는 무엇인가?
  3. 스택과 큐의 가장 큰 차이는 무엇인가?
  4. 올바른 괄호 문제에서 닫는 괄호가 나왔는데 스택이 비어 있으면 어떤 의미인가?
  5. 구간 합 문제에서 누적합을 사용하면 질문 하나를 몇 초가 아니라 어떤 시간복잡도로 처리할 수 있는가?

11. 핵심 정리

  1. O(1)은 입력 크기와 관계없이 일정한 시간입니다.
  2. O(N)은 입력 크기만큼 반복하는 알고리즘입니다.
  3. O(N log N)은 정렬에서 자주 등장하는 효율적인 시간복잡도입니다.
  4. 스택은 나중에 들어온 것이 먼저 나가는 LIFO 구조입니다.
  5. 큐는 먼저 들어온 것이 먼저 나가는 FIFO 구조입니다.
  6. 올바른 괄호 쌍 문제는 스택으로 해결할 수 있습니다.
  7. 큐 회전 문제는 앞에서 꺼낸 값을 뒤에 다시 넣는 방식으로 해결합니다.
  8. 구간 합 문제는 누적합을 사용하면 질문 하나를 O(1)에 처리할 수 있습니다.
  9. 입력이 많을 때는 C에서는 scanf/printf, Python에서는 sys.stdin.readline을 활용합니다.
  10. 중등정보올림피아드에서는 시간복잡도 분석이 풀이의 출발점입니다.
