시간복잡도 (Time Complexity)
// 알고리즘이 입력 크기 n에 따라 얼마나 많은 연산을 수행하는가
시간복잡도란 입력 크기 n이 커질수록 알고리즘 실행 시간이 어떻게 증가하는지를 나타내는 지표입니다. Big-O 표기법을 사용해 최악의 경우(Worst Case)를 기준으로 표현합니다.
예: O(n²) → 입력이 2배 → 연산이 4배 증가

🔍 n 값에 따른 연산 횟수 계산기
배열 (Array)
// 연속된 메모리 공간에 같은 타입의 데이터를 순서대로 저장
배열은 인덱스(index)로 임의 접근이 가능한 자료구조입니다. arr[i] 접근은 O(1)이지만, 중간 삽입/삭제는 O(n)이 걸립니다. C언어에서 배열은 int arr[5] = {10, 20, 30, 40, 50}; 형태로 선언합니다.
🎮 배열 시뮬레이터 (최대 8칸)
✅ 장점
  • arr[i] — O(1) 임의 접근
  • 메모리 공간이 연속적 (캐시 효율 ↑)
  • 구현이 단순, 빠른 순차 탐색
❌ 단점
  • 크기 고정 (정적 배열)
  • 중간 삽입·삭제 — O(n) (이동 필요)
  • 메모리 낭비 가능성
📝 C언어 배열 예시
스택 (Stack)
// LIFO — Last In, First Out (후입선출)
스택은 책을 쌓는 것처럼 가장 나중에 넣은 것이 먼저 나오는 구조입니다. push() — 맨 위에 추가, pop() — 맨 위 제거, peek() — 맨 위 확인. 모든 연산이 O(1)입니다. 활용: 함수 호출 스택, 괄호 검사, DFS, 역순 출력.
🎮 스택 시뮬레이터
스택이 비어있습니다
📝 C언어 스택 구현 (배열 기반)
큐 (Queue)
// FIFO — First In, First Out (선입선출)
큐는 줄 서기처럼 먼저 들어온 것이 먼저 나오는 구조입니다. enqueue() — 뒤에 추가, dequeue() — 앞에서 제거. 모든 연산이 O(1)입니다. 활용: BFS, 프린터 스풀, 프로세스 스케줄링.
🎮 큐 시뮬레이터
← DEQUEUE (Front) ENQUEUE (Rear) →
큐가 비어있습니다
📝 C언어 큐 구현 (원형 배열 기반)
연결 리스트 (Linked List)
// 노드들이 포인터로 연결된 동적 자료구조
각 노드가 data와 다음 노드를 가리키는 next 포인터를 가집니다. 삽입·삭제가 O(1)(위치 알 때)이지만, i번째 접근은 O(n)입니다. 배열과 달리 동적으로 크기 변경이 가능합니다.
🎮 단일 연결 리스트 시뮬레이터
리스트가 비어있습니다 (HEAD → NULL)
