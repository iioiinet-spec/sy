예시 답변 (이진탐색 연결)
1. 책이 번호순으로 정렬되어 있다면: → 중간 번호 확인 → 찾는 책 번호가 더 크면 오른쪽 절반만 탐색 → 작으면 왼쪽 절반만 탐색 → 반복 (이진 탐색 - 이분 탐색) 2. 정렬이 안 되어 있다면: → 처음부터 끝까지 하나씩 확인 → (선형 탐색 - 비효율적) 결론: 데이터를 미리 정렬해두면 탐색이 훨씬 빨라집니다. 이것이 '자료구조와 알고리즘'이 중요한 이유입니다.

알고리즘버블 정렬과 선택 정렬의 차이를 설명해보세요.▼
버블 정렬: 인접한 두 원소를 비교해 큰 것을 뒤로 보냄 [5, 3, 1] → [3, 5, 1] → [3, 1, 5] → [1, 3, 5] 특징: 매 번 비교·교환 → 교환 횟수 많음 선택 정렬: 전체에서 가장 작은 값을 찾아 맨 앞과 교환 [5, 3, 1] → 최솟값 1 찾기 → [1, 3, 5] 특징: 비교는 많지만 교환 횟수는 적음 공통점: 둘 다 O(n²) - 데이터가 많아지면 느려짐 차이: 버블은 교환이 많고, 선택은 교환이 적음

알고리즘재귀(Recursion)란 무엇이고, 언제 쓰면 좋을까요?▼
예시 답변
재귀는 함수가 자기 자신을 호출하는 방법입니다. 반드시 종료 조건(기저 조건)이 있어야 무한 반복을 막을 수 있습니다.
팩토리얼 예시: 5! = 5 × 4! 4! = 4 × 3! ... (계속) 1! = 1 ← 기저 조건(여기서 멈춤) 재귀가 유리한 경우: - 트리 구조 탐색 (폴더 안의 폴더) - 분할 정복 (큰 문제 → 작은 문제로 분해) - 피보나치, 하노이탑
주의: "재귀는 느릴 수 있다. 중복 계산이 생기면 메모이제이션으로 개선할 수 있다"고 언급하면 가산점!
알고리즘스택(Stack)과 큐(Queue)의 차이와 실생활 예를 들어보세요.▼
스택 (Stack) - LIFO (Last In, First Out) 마지막에 넣은 것이 먼저 나옴 실생활 예: 접시 쌓기, 뒤로가기 버튼, 괄호 검사 활용: 함수 호출 스택, 수식 계산 큐 (Queue) - FIFO (First In, First Out) 먼저 넣은 것이 먼저 나옴 실생활 예: 줄 서기, 프린터 대기열, 콜센터 대기 활용: BFS(너비 우선 탐색), 작업 스케줄링
