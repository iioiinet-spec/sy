단조 스택 인터렉티브 해설

소의 키가 하나씩 들어올 때 스택에서 누가 빠지고, 정답이 왜 증가하는지 직접 확인하는 수업용 HTML 자료입니다.

문제 이해 핵심 아이디어 시뮬레이터 정답 코드 예제 추적 시간복잡도 핵심 정리

1. 문제 이해

소들이 한 줄로 서 있고 모두 오른쪽을 보고 있습니다. 각 소는 오른쪽에 있는 소들 중에서 자신보다 키가 작은 소를 볼 수 있습니다. 하지만 중간에 자신보다 크거나 같은 소가 나오면 그 뒤는 볼 수 없습니다.

예시

키: 5 2 4 2 6 1

키가 5인 소는 오른쪽의 2, 4, 2를 볼 수 있지만 6이 나오면 시야가 막힙니다.

N이 최대 80000 수준이면 이중 반복문 O(N²)은 시간초과가 납니다. 단조 스택으로 O(N)에 해결해야 합니다.

2. 핵심 아이디어: 관점 바꾸기

어려운 관점각 소가 오른쪽으로 몇 마리를 볼 수 있을까?
쉬운 관점현재 들어온 소를 볼 수 있는 왼쪽 소가 몇 마리일까?

2.1 스택에 남기는 것

스택에는 앞으로도 오른쪽의 소를 볼 가능성이 있는 소들만 남깁니다.

while (top >= 0 && stack[top] <= height) {
    top--;
}

현재 소보다 작거나 같은 소는 현재 소에게 시야가 막히므로 스택에서 제거합니다.

제거가 끝난 뒤 스택에 남은 소들은 모두 현재 소보다 큽니다. 따라서 그 소들은 현재 소를 볼 수 있습니다.

3. 인터렉티브 단조 스택 시뮬레이터

키 배열을 입력하고 한 단계씩 눌러 보세요. 기본 예시는 5 2 4 2 6 1입니다.

Monotonic Stack Simulator

4. 정답 C 코드

#include <stdio.h>

#define MAX 80001

int stack[MAX];
int top = -1;

int main() {
    int N;
    long long answer = 0;

    scanf("%d", &N);

    for (int i = 0; i < N; i++) {
        int height;
        scanf("%d", &height);

        while (top >= 0 && stack[top] <= height) {
            top--;
        }

        answer += top + 1;

        stack[++top] = height;
    }

    printf("%lld\n", answer);

    return 0;
}
정답은 int 범위를 넘을 수 있으므로 반드시 long long을 사용합니다.

6. 시간복잡도

각 소는 스택에 한 번 들어가고, 많아야 한 번 제거됩니다.

작업횟수결과
스택 삽입소마다 1번O(N)
스택 삭제소마다 최대 1번O(N)
전체삽입 + 삭제O(N)
단조 스택은 겉으로는 while문이 있지만, 전체적으로 보면 각 원소가 한 번만 들어가고 한 번만 빠지므로 O(N)입니다.

7. 핵심 정리

  1. 이 문제는 단조 스택 문제입니다.
  2. 스택에는 앞으로 오른쪽 소를 볼 가능성이 있는 소들만 남깁니다.
  3. 현재 소보다 작거나 같은 소는 스택에서 제거합니다.
  4. 제거 후 스택에 남은 소의 수가 현재 소를 볼 수 있는 왼쪽 소의 수입니다.
  5. answer에는 매번 스택의 크기 top + 1을 더합니다.
  6. 정답이 커질 수 있으므로 long long을 사용합니다.
  7. 전체 시간복잡도는 O(N)입니다.
