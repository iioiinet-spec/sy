우선순위 큐와 이진 힙

우선순위 큐의 추상 자료형, 완전 이진 트리에 기반한 최대 힙·최소 힙, 힙의 삽입·삭제 원리, 중앙값 실시간 추적, 카드 정렬 최소 비용 알고리즘을 정리한 강의 노트입니다.

2. 우선순위 큐(Priority Queue)

2.1 일반 큐와 우선순위 큐의 차이

일반 큐 먼저 들어온 데이터가 먼저 나갑니다.
FIFO 구조입니다.
우선순위 큐 먼저 들어온 순서가 아니라 우선순위가 높은 데이터가 먼저 나갑니다.

예시

입력: (1, A), (5, B), (3, C)

출력 순서: B → C → A

우선순위가 가장 높은 5가 먼저 나옵니다.

2.2 우선순위 큐가 필요한 문제

  1. 가장 작은 값 빠르게 찾기
  2. 가장 큰 값 빠르게 찾기
  3. 실시간 중앙값 찾기
  4. 최소 비용 계산
  5. 다익스트라 알고리즘
  6. 작업 스케줄링

2.3 우선순위 큐의 시간복잡도

연산 시간복잡도 설명
삽입O(log N)힙의 높이만큼 이동
삭제O(log N)루트 삭제 후 재정렬
최댓값/최솟값 확인O(1)루트만 확인

3. 힙(Heap)이란?

힙은 우선순위 큐를 구현하는 대표적인 자료구조입니다. 힙은 완전 이진 트리를 기반으로 하며, 부모와 자식 사이에 일정한 크기 관계를 유지합니다.

최대 힙 부모 노드가 자식 노드보다 크거나 같습니다.
루트에는 최댓값이 있습니다.
최소 힙 부모 노드가 자식 노드보다 작거나 같습니다.
루트에는 최솟값이 있습니다.
힙은 전체가 완전히 정렬된 구조가 아닙니다. 루트가 최댓값 또는 최솟값이라는 것이 핵심입니다.

4. 완전 이진 트리

4.1 완전 이진 트리의 특징

완전 이진 트리는 위쪽 레벨부터 채워지고, 같은 레벨에서는 왼쪽부터 차례대로 채워지는 트리입니다.

10 / \ 8 5 / \ / 3 2 1

4.2 왜 배열로 구현할 수 있을까?

완전 이진 트리는 중간에 빈칸 없이 왼쪽부터 채워지므로 배열에 저장하기 쉽습니다. 포인터로 트리를 만들지 않아도 인덱스 계산만으로 부모와 자식을 찾을 수 있습니다.

5. 힙의 인덱스 규칙

힙을 배열로 구현할 때 보통 1번 인덱스부터 사용합니다. 그러면 부모와 자식 인덱스를 쉽게 계산할 수 있습니다.

관계 공식
부모i / 2
왼쪽 자식i * 2
오른쪽 자식i * 2 + 1

배열:

index : 1  2  3  4  5  6
value : 50 30 20 10 5 15

트리 구조:

50 / \ 30 20 / \ / 10 5 15
힙 구현에서 가장 많이 발생하는 실수는 자식 인덱스가 배열 범위를 넘었는지 확인하지 않는 것입니다.

6. 최대 힙(Max Heap)

최대 힙은 부모 노드가 자식 노드보다 크거나 같은 구조입니다. 따라서 항상 루트에 가장 큰 값이 있습니다.

100 / \ 50 30 / \ / \ 20 10 5 1
최대 힙 규칙: 부모 ≥ 자식

7. 최대 힙 삽입 원리

7.1 삽입 과정

  1. 새 값을 배열의 맨 뒤에 넣습니다.
  2. 부모와 비교합니다.
  3. 부모보다 크면 부모와 교환합니다.
  4. 힙 조건이 만족될 때까지 위로 올라갑니다.

7.2 삽입 예시

현재 힙:

50 / \ 30 20

40 삽입:

50 / \ 30 20 / 40

40은 부모 30보다 크므로 교환합니다.

50 / \ 40 20 / 30
삽입 시간복잡도: O(log N)

7.3 최대 힙 삽입 C 코드

#include <stdio.h>

#define MAX 100001

int heap[MAX];
int size = 0;

void push(int value) {
    heap[++size] = value;

    int current = size;

    while (current > 1) {
        int parent = current / 2;

        if (heap[parent] >= heap[current]) {
            break;
        }

        int temp = heap[parent];
        heap[parent] = heap[current];
        heap[current] = temp;

        current = parent;
    }
}

int main() {
    push(10);
    push(50);
    push(30);
    push(70);

    for (int i = 1; i <= size; i++) {
        printf("%d ", heap[i]);
    }

    return 0;
}

8. 최대 힙 삭제 원리

8.1 삭제 대상

최대 힙에서 삭제되는 값은 항상 루트, 즉 가장 큰 값입니다.

8.2 삭제 과정

  1. 루트 값을 저장합니다.
  2. 마지막 노드를 루트로 옮깁니다.
  3. 자식 노드들과 비교합니다.
  4. 더 큰 자식과 교환합니다.
  5. 힙 조건이 만족될 때까지 아래로 내려갑니다.

8.3 삭제 예시

현재 힙:

100 / \ 50 30 / \ 20 10

100 삭제 후 마지막 값 10을 루트로 이동:

10 / \ 50 30 / 20

50과 교환:

50 / \ 10 30 / 20

20과 교환:

50 / \ 20 30 / 10
삭제 시간복잡도: O(log N)

8.4 최대 힙 삭제 C 코드

#include <stdio.h>

#define MAX 100001

int heap[MAX];
int size = 0;

void push(int value) {
    heap[++size] = value;

    int current = size;

    while (current > 1) {
        int parent = current / 2;

        if (heap[parent] >= heap[current]) {
            break;
        }

        int temp = heap[parent];
        heap[parent] = heap[current];
        heap[current] = temp;

        current = parent;
    }
}

int pop() {
    int result = heap[1];
    heap[1] = heap[size--];

    int current = 1;

    while (1) {
        int left = current * 2;
        int right = current * 2 + 1;
        int largest = current;

        if (left <= size && heap[left] > heap[largest]) {
            largest = left;
        }

        if (right <= size && heap[right] > heap[largest]) {
            largest = right;
        }

        if (largest == current) {
            break;
        }

        int temp = heap[current];
        heap[current] = heap[largest];
        heap[largest] = temp;

        current = largest;
    }

    return result;
}

int main() {
    push(10);
    push(50);
    push(30);
    push(70);

    printf("%d\n", pop());
    printf("%d\n", pop());

    return 0;
}

9. 최소 힙(Min Heap)

최소 힙은 부모 노드가 자식 노드보다 작거나 같은 구조입니다. 따라서 항상 루트에 가장 작은 값이 있습니다.

1 / \ 3 5 / \ / \ 7 8 9 10
최소 힙 규칙: 부모 ≤ 자식

9.1 최대 힙과 최소 힙의 차이

구분 규칙 루트
최대 힙부모 ≥ 자식최댓값
최소 힙부모 ≤ 자식최솟값

9.2 C++ STL priority_queue

// 최대 힙
priority_queue<int> pq;

// 최소 힙
priority_queue<int, vector<int>, greater<int>> pq;

10. 실전 문제: 가운데를 말해요

10.1 문제 설명

숫자가 하나씩 입력될 때마다 현재까지 입력된 숫자들의 중앙값을 출력하는 문제입니다.

입력 순서: 1, 5, 2, 10

출력 중앙값: 1, 1, 2, 2

10.2 핵심 아이디어

힙 두 개를 사용합니다.

힙 역할 의미
최대 힙작은 값들 관리중앙값 이하
최소 힙큰 값들 관리중앙값 이상

10.3 중앙값 유지 규칙

  1. 최대 힙 크기는 최소 힙 크기와 같거나 1개 더 많게 유지합니다.
  2. 최대 힙의 루트는 최소 힙의 루트보다 작거나 같아야 합니다.
  3. 중앙값은 최대 힙의 루트입니다.

10.4 진행 예시

1 입력

최대 힙: 1
최소 힙: 없음
중앙값: 1

5 입력

최대 힙: 1
최소 힙: 5
중앙값: 1

2 입력

최대 힙: 2 1
최소 힙: 5
중앙값: 2
실시간으로 중앙값을 구할 때 매번 정렬하면 느립니다. 두 개의 힙을 사용하면 매 입력마다 O(log N)에 처리할 수 있습니다.

11. 실전 문제: 카드 정렬하기

11.1 문제 설명

여러 카드 묶음이 있을 때, 두 묶음을 합치는 비용은 두 묶음 크기의 합입니다. 모든 카드를 하나로 합칠 때 필요한 최소 비용을 구합니다.

11.2 핵심 아이디어

항상 가장 작은 두 묶음을 먼저 합쳐야 전체 비용이 최소가 됩니다. 따라서 최소 힙을 사용합니다.

예시: 10, 20, 40

잘못된 방법:

40 + 20 = 60
60 + 10 = 70
총 비용 = 130

최적 방법:

10 + 20 = 30
30 + 40 = 70
총 비용 = 100

11.3 C++ STL 코드

#include <iostream>
#include <queue>
#include <vector>
#include <functional>

using namespace std;

int main() {
    priority_queue<int, vector<int>, greater<int>> pq;

    int N;
    cin >> N;

    for (int i = 0; i < N; i++) {
        int x;
        cin >> x;
        pq.push(x);
    }

    int answer = 0;

    while (pq.size() > 1) {
        int a = pq.top();
        pq.pop();

        int b = pq.top();
        pq.pop();

        int sum = a + b;
        answer += sum;

        pq.push(sum);
    }

    cout << answer;
    return 0;
}

11.4 C언어 최소 힙 코드 핵심

void push(int value) {
    heap[++size] = value;
    int current = size;

    while (current > 1) {
        int parent = current / 2;

        if (heap[parent] <= heap[current]) break;

        int temp = heap[parent];
        heap[parent] = heap[current];
        heap[current] = temp;

        current = parent;
    }
}
카드 정렬하기 시간복잡도: O(N log N)

13. 핵심 정리

  1. 우선순위 큐는 우선순위가 높은 데이터가 먼저 나오는 자료구조입니다.
  2. 힙은 우선순위 큐를 구현하는 대표 자료구조입니다.
  3. 힙은 완전 이진 트리를 기반으로 합니다.
  4. 최대 힙은 부모가 자식보다 크거나 같습니다.
  5. 최소 힙은 부모가 자식보다 작거나 같습니다.
  6. 힙은 배열로 구현할 수 있습니다.
  7. 부모 인덱스는 i / 2입니다.
  8. 왼쪽 자식 인덱스는 i * 2입니다.
  9. 오른쪽 자식 인덱스는 i * 2 + 1입니다.
  10. 힙 삽입은 새 값을 맨 뒤에 넣고 위로 올리는 과정입니다.
  11. 힙 삭제는 루트를 제거하고 마지막 값을 루트로 올린 뒤 아래로 내리는 과정입니다.
  12. 힙의 삽입과 삭제 시간복잡도는 O(log N)입니다.
  13. 가운데를 말해요 문제는 최대 힙과 최소 힙 두 개를 사용합니다.
  14. 카드 정렬하기 문제는 최소 힙을 사용하는 대표적인 최소 비용 문제입니다.
  15. 힙은 중등정보올림피아드 실전 문제에서 매우 자주 등장합니다.
